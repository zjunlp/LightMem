## 目标
- 仓库中所有文本统一存储为 `LF`
- 开发者本地按系统自动换行（Windows `CRLF`，Linux/Mac `LF`）
- 清理并修复因换行不一致产生的警告，规范已暂存内容

## 新增文件
- 在仓库根目录新增 `.gitattributes`：
```
* text=auto
```
- 如需为少数脚本固定工作区行尾，可按需扩展（可选）：
```
*.sh text eol=lf
*.bat text eol=crlf
*.cmd text eol=crlf
*.ps1 text eol=crlf
```

## 开发者本地 Git 配置
- Windows：
  - `git config --global core.autocrlf true`
  - `git config --global safecrlf warn`
- Linux/Mac：
  - `git config --global core.autocrlf input`
  - `git config --global safecrlf warn`

## 修复已暂存文件的换行符
- 提交 `.gitattributes` 后进行规范化：
  - 首选快速方式：`git add --renormalize .`
  - 若仍有异常，执行彻底规范化：
    - `git rm -r --cached .`
    - `git add --renormalize .`
- 提交规范化结果：`git commit -m "Add .gitattributes and normalize EOLs"`

## 验证与消除警告
- `git status` 应无多余改动；
- `git diff --check` 检查是否还有行尾/空白问题；
- 在 Windows 上检查工作区文件为 `CRLF`，在 Linux/Mac 上为 `LF`（仓库中统一为 `LF`）。

## 注意事项
- `.gitattributes` 的 `* text=auto` 会自动跳过二进制文件，无需逐一标注；
- 若项目中存在必须保留特定行尾的特殊文件，可按需添加精确规则；
- 规范化会触发一次较大的索引更新，建议在独立分支完成并合并。
